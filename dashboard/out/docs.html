Getting Started

The Architect-Extractor Pattern

Parserator uses a revolutionary two-stage AI process that achieves 95% accuracy while reducing token costs by 70%:

  1. Architect LLM: Analyzes a sample and creates a detailed parsing plan
  2. Extractor LLM: Executes the plan on your full dataset

Quick Start

1. Install the SDK

npm install parserator-sdk

2. Get your API key

Visit the Dashboard to create your API key.

3. Make your first request

import { Parserator } from 'parserator-sdk';

const parser = new Parserator('your-api-key');

const result = await parser.parse({
  inputData: `
    John Smith
    Email: john@example.com
    Phone: (555) 123-4567
    Address: 123 Main St, Boston, MA
  `,
  outputSchema: {
    name: "string",
    email: "string", 
    phone: "string",
    address: "string"
  }
});

console.log(result.parsedData);
// {
//   name: "John Smith",
//   email: "john@example.com",
//   phone: "(555) 123-4567", 
//   address: "123 Main St, Boston, MA"
// }

API Reference

Base URL

https://app-5108296280.us-central1.run.app

POST /v1/parse

Parse unstructured data into structured JSON using AI.

Request Body

{
  "inputData": "string",      // Raw unstructured data to parse
  "outputSchema": "object",   // Desired JSON structure
  "instructions": "string"    // Optional: Additional parsing context
}

Response

{
  "success": true,
  "parsedData": {
    // Your structured data
  },
  "metadata": {
    "architectPlan": {
      // The parsing plan created by the Architect LLM
    },
    "confidence": 0.95,
    "tokensUsed": 1250,
    "processingTimeMs": 800
  }
}

Error Response

{
  "success": false,
  "error": {
    "code": "INVALID_SCHEMA",
    "message": "Output schema is invalid",
    "details": "Additional error information"
  }
}

Authentication

All API requests require an API key. Include your key in the Authorization header: 

curl -X POST https://app-5108296280.us-central1.run.app/v1/parse \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "inputData": "...",
    "outputSchema": {...}
  }'

Keep your API key secure! Never expose it in client-side code or public repositories.
